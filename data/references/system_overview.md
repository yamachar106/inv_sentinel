# inv_kuroten 投資シグナルシステム 全体像

## このシステムは何か

毎日自動で日米株式市場を10種類のロジックでスキャンし、**「今買うべき株」と「今売るべき株」**をSlackに通知するシステム。

複数のロジックが同時に「買い」と言っている銘柄ほど高い確信度でシグナルを出し、相場環境（BULL/NEUTRAL/BEAR）に応じてポジションサイズを自動調整する。

---

## 日々の運用フロー

```
[毎日 16:00以降]  daily_run.py 自動実行（GitHub Actions）
       ↓
  10種のロジックが日米株をスキャン
       ↓
  シグナル発火 → Slackチャンネルに通知
       ↓
[翌朝 08:00]  朝リマインド「今日やること」
       ↓
  寄付で売買実行（手動）
```

### Tier 2: 補強戦略（確度を上げるフィルタ）

単独でエントリーするのではなく、Tier 1のシグナルと重なった時に確信度を上げる役割。

#### 6. VCP（ボラティリティ収縮パターン）

| 項目 | 内容 |
|------|------|
| **対象** | JP/US ブレイクアウト候補 |
| **ロジック** | ブレイクアウト前に値動きが収縮していくパターン（Minervini手法）を検出 |
| **役割** | ブレイクアウトシグナルに `vcp_detected` フラグを付与 → 確信度+1 |
| **判定基準** | 2-6回の収縮、各回が前回比60%以下、収縮中に出来高枯渇 |
| **実行** | 毎日（BOスキャンと同時） |

**使い方**: 「BO + VCP」= 確信度2 → 通常より大きく買ってよい

---

#### 7. PEAD（決算サプライズドリフト）

| 項目 | 内容 |
|------|------|
| **対象** | JP MEGA ユニバース |
| **ロジック** | 決算の進捗率が例年レンジを大幅に超えている銘柄を検出（サプライズ20%以上） |
| **期待値** | +2.6〜9.4%/四半期（Ball & Brown 1968） |
| **保有期間** | 60営業日 |
| **実行** | 月曜 × 決算月（1,2,4,5,7,8,10,11月）のみ |

**使い方**: 他のシグナルと重なれば確信度アップ。単独でも+20%超サプライズならエントリー検討。

### Tier 3: フレームワーク（判断支援）

#### 11. コンフルエンス（シグナル重畳スコアリング）

複数のロジックが同じ銘柄に対して同時にシグナルを出した場合、確信度を上げる仕組み。

```
確信度1 (LOW)       ← ロジック1つだけ     → BULL時のみエントリー
確信度2 (MODERATE)  ← 2つ重畳            → 通常サイズで買い
確信度3 (HIGH)      ← 3つ重畳            → 大きめに買い
確信度4 (VERY HIGH) ← 4つ以上            → フルサイズ
```

**シグナル分類**:
- テクニカル系: breakout, vcp, stage2, mega_bo
- ファンダメンタル系: kuroten, pead, earnings_accel, insider

テクニカル + ファンダが両方揃うと最も信頼性が高い。

---

#### 12. レジーム適応型ポジションサイジング

相場環境（BULL/NEUTRAL/BEAR）に応じて自動でポジションサイズを調整。

| 環境 | 判定基準 | サイズ | 最低確信度 | 最大エクスポージャー |
|------|---------|:---:|:---:|:---:|
| **BULL** | 株価 > SMA50 > SMA200 | 半ケリー | 1 | 90% |
| **NEUTRAL** | 中間 | 1/4ケリー | 2 | 70% |
| **BEAR** | 株価 < SMA200 < SMA50 | 1/8ケリー | 3 | 40% |

**BEARでは確信度3未満のシグナルは全て無視** = 弱い根拠では絶対に買わない。

## 売却ルール（6つ）

保有中の全ポジションを毎日自動チェック。該当すればSlackに通知。

| 優先度 | ルール | 緊急度 | 条件 |
|:---:|--------|:---:|------|
| 1 | 利確目標達成 | 🔴 HIGH | 黒字転換: 2倍 / BO: +20% / MEGA: +40% |

| 3 | トレーリングストップ | 🔴 HIGH | +80%到達後、高値から-20%下落 |
| 4 | 損切り | 🔴 HIGH | 黒字転換-25% / BO-10% / MEGA-20% |
| 5 | 保有期間超過 | 🟡 MEDIUM | 2年（30日前から警告） |
| 6 | Stage 3/4警告 | 🟡/🔴 | 30週MAが天井/下降ステージ |

## コマンド一覧

```bash
# 通常運用（自動実行されるのでほぼ触らない）
python daily_run.py                    # 全戦略実行
python daily_run.py --dry-run          # 通知なしプレビュー
python daily_run.py --strategy jp-mega # JP MEGAのみ
python daily_run.py --strategy monitor # ポジション監視のみ

# 期待値確認
python -m screener.expected_value          # 全戦略EV一覧
python -m screener.expected_value --detail # 詳細（R:R, Kelly等）

# ポジション管理
python -m screener.portfolio list                   # 全ポジション
python -m screener.portfolio add 7974 --strategy mega_jp --buy-date 2026-04-13 --buy-price 10000 --shares 100
python -m screener.portfolio remove 7974 --sell-price 14000 --sell-reason "TP達成"

# バックテスト
python backtest_breakout.py --codes AAPL,NVDA       # BO戦略
```
